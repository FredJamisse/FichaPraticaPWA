1. O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

    Resposta: “Commit” é uma operação que serve para guardar as alterações feitas no código; essas alterações vão sempre com uma descrição. “Push” é uma operação que serve para enviar o código para o repositório online. “Pop” é uma operação que serve para aplicar de volta alterações guardadas temporariamente (via stash) antes de fazer commit e push.

  2. Quais são as vantagens de usar Branches num repositório Git?

    Resposta: Usar Branches num repositório Git é vantajoso porque permite fazer alterações sem comprometer o código principal.

  3. Nos merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?

    Resposta: O conflito pode ocorrer quando duas versões de um mesmo ficheiro têm alterações incompatíveis nas mesmas linhas. Para esses casos, a forma de resolver é escolhendo manualmente a versão a considerar.

  4. “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?

    Resposta: A diferença entre Clone e Fork é que, enquanto o Clone permite copiar um repositório online para ser usado localmente, o Fork cria uma cópia do repositório na tua conta online.
